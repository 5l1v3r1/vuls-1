-deep option

Deep scan modeの設定のチェックを行うためには、 vuls configtest --deep する必要がある。

Deep Scan modeの詳細は以下を参照すること。

  • Architecture/Deep Scan
  • Configtest/Deep Scan Mode

-ssh-native-insecure option

Vulsは複数のSSH接続方法をサポートしている。

デフォルトでは、外部SSHコマンドが使われる。 GoのライブラリではサポートしていないChperアルゴリズムやProxyCommandを使いたい場合に外部SSHコマンドを利用すると良い。スキャン対象サーバの/etc/sudoersに以下の行を追加しておくことを忘れないように。 (username: vuls)

Defaults:vuls !requiretty

GoライブラリのSSH実装であるcrypto/sshを使いたいバイアはこのオプションを指定する。外部SSHコマンドが存在しない場合に使うと良い。 ただし、このモードではHostKeyをチェックしないのでMITM攻撃を受ける可能性があることに注意するべし。

-ask-key-password option

SSH key password-ask-key-password
empty password-
with passwordrequiredor use ssh-agent

Example: Config Fileに定義されたすべてのサーバをスキャンする

$ vuls scan -ask-key-password

このコマンドが意味することは...

  • スキャン時にSSH秘密鍵のパスワードを入力する
  • 設定ファイルに定義された全サーバをスキャンする

Example: 設定フィアルに定義されたサーバのうち特定のサーバをスキャンする

$ vuls scan server1 server2

このコマンドが意味することは...

  • SSH鍵認証、パスワードが設定されてない秘密鍵を用いる（-ask-key-passwordオプションを指定していないので）
  • 2台のみスキャンする（server1, server2)

Example: SSH経由ではなく、コマンドを直接実行してスキャンする

設定ファイルのhostに、localhst or 127.0.0.1が指定されている、かつ、portにlocalが指定されているサーバは、SSH経由ではなく直接コマンドを発行してスキャンする。詳細は、Architecture section参照。

  • config.toml

    [servers]

[servers.localhost]
host         = "localhost" # or "127.0.0.1"
port         = "local"

Example: Scan containers (Docker/LXD/LXC)

コンテナはSSHデーモンを起動しないで運用するのが一般的である。Docker Blog:Why you don't need to run SSHd in your Docker containers

Docker

Vulsがコンテナをスキャンするときは、SSHではなく、docker execでコンテナに対してコマンドを発行してスキャンする。 詳細はArchitecture section参照。

If you don’t want to use root, create a Unix group called docker and add users to it For details, see https://docs.docker.com/install/linux/linux-postinstall/

  • 実行中の全コンテナをスキャンする場合は、"${running}" を設定ファイルに指定する。 ``` [servers]

    [servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"

[servers.172-31-4-82.containers]
includes = ["${running}"]
```

  • ある特定のコンテナのみをスキャンする場合は、設定ファイルにスキャンしたいコンテナの、コンテナのIDかコンテナ名を記述する。 次のサンプルでは、 container_name_a と、 4aa37a8b63b9のみスキャンされる。 スキャン前に指定したコンテナが起動中か確認すること。 もしも指定されたコンテナが実行中ではない場合、エラーメッセージを表示してスキャンは終了する。 ``` [servers]

    [servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"

[servers.172-31-4-82.containers]
includes = ["container_name_a", "4aa37a8b63b9"]
```

  • 特定のコンテナ以外をスキャンする場合 ``` [servers]

    [servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"

[servers.172-31-4-82.containers]
includes = ["${running}"]
excludes = ["container_name_a", "4aa37a8b63b9"]
```

  • コンテナのみをスキャンする場合（ホストはスキャンしない）

    • --containers-only option is available.

LXD

Vuls scans lxd via lxc exec instead of SSH.

[servers]

[servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"

[servers.172-31-4-82.containers]
type = "lxd"
includes = ["${running}"]

LXC

Vuls scans lxc via lxc-attach instead of SSH.

[servers]

[servers.172-31-4-82]
host         = "172.31.4.82"
user        = "ec2-user"
keyPath     = "/home/username/.ssh/id_rsa"

[servers.172-31-4-82.containers]
type = "lxc"
includes = ["${running}"]

LXC required root privilege.

Example of /etc/sudoers on target servers

vuls ALL=(ALL) NOPASSWD:/usr/bin/lxc-attach -n *, /usr/bin/lxc-ls *
